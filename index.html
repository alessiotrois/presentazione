Definition of processing levels (Instrument)

Preliminary description of scientific data format for IXPE: LV0a, LV0b, LV1a v0.1_201706

The telemetry parser (LV0->LV0a) consists in the rewriting of the raw telemetry data produced by the Instrument (LV0) in more suitable format with the data divided according to type, mode, detector, etc... No information is added: it is only a translation/reorganization of the available data. The format of the output files will be the widely used FITS standard. This task populates the LV0a archive with all the telemetry produced by the Instrument(Scientific Data, Housekeeping's, Scientific Ratemeters, Telemetry Reports, etcetc).

The LV0 Correction (LV0a->LV1) consists of a set of programs that will process LV0a data producing data structures and formats optimized for the following data analysis steps. The LV0a data produced by the telemetry parser are a simple translation in FITS format of the telemetry content. Subsequent analysis requires that some of the data be processed and stored with some added information.
For example: conversion of units, derivation of new quantities which are a function of data stored in different files/columns of LV0a data, uniform the different time formats in MET (Mission Elapsed Time), stops any unnecessary engineering information, etc.etc.
All the corrections which are relatively stable and do not depend on time variable calibration parameters are done in the LV0corr subsystem.

Definition of processing levels (SOC)

IXPE Science Data Management Plan DRAFT version 2017-Aug- 04

Level 0 (L0) data are raw CCSDS telemetry packets collected by the MOC and packaged into 24- hour packet files. These files along with pass reports are then transferred to the SOC. Housekeeping data from both the spacecraft and instrument, instrument science data, metrology-system data, and star-tracker data are included.

Level-1 (L1) data contains everything: the telemetry packets have been unpacked, time- ordered, any duplicates deleted, and the results converted into FITS files. The L1 data are unfiltered; thus, for example, event files will include all events in a given pointing (or slew), which may include data from the source, data in the South Atlantic Anomaly (SAA), data when pointed at the Earth, etc.

Level-2 (L2) data contain clean data that have been time filtered to include only data from Good Time Intervals (GTI) when the detector was exposed to the source. In addition, relevant calibrations have been applied: Positional data will have been converted to J2000 sky coordinates, etc. To reduce the size of downloads, the L2 data files will include the initial track direction but will not contain track images.
